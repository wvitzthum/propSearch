# UI/UX Quality Assurance Audit: 2026-03-08 (E)

## Executive Summary
The propSearch dashboard has reached significant functional maturity, with high-density data views, financial modeling, and lead ingestion pipelines operational. However, **Map spatial intelligence is currently compromised** due to aggressive CSS filters that suppress environmental context (Rivers, Parks, Streets). Additionally, the `floor_level` requirement is pending implementation across the data and frontend layers.

## 1. Map Usability & Spatial Intelligence (CRITICAL)
| Requirement | Status | Finding |
| --- | --- | --- |
| Semi-transparent Metro Overlay | **FAIL** | Lines are too thin (3px) and transparent (0.45), making them invisible against the dark background. |
| Carto Dark Matter Tiles | **FAIL** | `index.css` filters (`brightness(0.7) contrast(1.4)`) crush the low-end spectrum, rendering the Thames and Hyde Park as pure black. |
| Environmental Shading | **FAIL** | Rivers and Parks are indistinguishable from the background. |

**Remediation:**
- Remove `filter` from `.retro-map .leaflet-container` in `index.css`.
- Update `MetroOverlay` in `Dashboard.tsx` to `weight: 4.5` and `opacity: 0.75`.
- Sync `LocationNodeMap.tsx` with these institutional standards.

## 2. Data Fidelity & Field Capture (PENDING)
- **Floor Level Intelligence:** The `floor_level` field is missing from `master.json` (DAT-028) and the corresponding UI components (FE-048).
- **Lease Depth:** Capture is consistent, but visualization in the Table view is missing.

## 3. Precision UX & Institutional Polish (MEDIUM)
- **Tooltip Methodology:** `REQUIREMENTS.md` specifies that tooltips must explain *how* metrics are calculated. Currently, most tooltips only provide definitions.
- **Vetted Status:** While `usePipeline.ts` supports the `vetted` state, there is no clear UI path to move an asset from `shortlisted` to `vetted`.

## 4. Visual Resilience (PASSED)
- `PropertyImage` component correctly handles 404/broken links with a themed fallback.
- High-res gallery implementation in `PreviewDrawer` and `PropertyDetail` is functional.

## Action Plan (Updated in Tasks.md)
1. **Critical:** Implement `MAP_USABILITY_GUIDE.md` (FE-047).
2. **High:** Audit and expand all Institutional Tooltips to include Methodology.
3. **Medium:** Implement `floor_level` display once DAT-028 is complete.
