## QA-027: UX Audit (Sidebar & Scroll Lock)

### 1. Property Detail Page Scroll Lock
- **Status:** **REPRODUCED / CONFIRMED**
- **Root Cause:** In `frontend/src/components/PreviewDrawer.tsx`, the `useEffect` that manages `document.body.style.overflow` sets it to `'hidden'` when the drawer is open. However, the cleanup function only removes the event listener and fails to reset the overflow to `'auto'`. If a user navigates from the Drawer to the Detail page (using the "Deep Asset Scan" link), the component unmounts while `isOpen` is true, leaving the body scroll locked.
- **Evidence:**
```tsx
// frontend/src/components/PreviewDrawer.tsx line 35
  useEffect(() => {
    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key === 'Escape') onClose();
    };
    if (isOpen) {
      document.body.style.overflow = 'hidden';
      window.addEventListener('keydown', handleKeyDown);
    } else {
      document.body.style.overflow = 'auto';
    }
    return () => window.removeEventListener('keydown', handleKeyDown); // BUG: No overflow reset here
  }, [isOpen, onClose]);
```

### 2. Sidebar Menu Flicker/Disappear
- **Status:** **INVESTIGATED**
- **Observation:** The sidebar uses `fixed` positioning with `backdrop-blur-xl` and `bg-linear-bg/80`. While no direct code bug makes it "disappear", the use of `backdrop-blur` on a fixed sidebar can cause rendering artifacts (flicker) in Chromium-based browsers when the main content (which also uses heavy gradients and noise overlays) scrolls underneath it.
- **Recommendation:** Add `backface-visibility: hidden;` and `transform: translateZ(0);` to the sidebar to promote it to its own GPU layer, reducing flicker during scroll.

## QA-028: Map Aesthetic Audit

### 1. Metro Line Visualization
- **Status:** **SUB-OPTIMAL**
- **Finding:** Metro lines are currently rendered at `weight: 1.5` with `opacity: 0.2`. This makes them nearly invisible against the dark Carto tiles, especially with the global CSS filters applied.
- **Requirement Gap:** `REQUIREMENTS.md` calls for a high-precision map. Current thickness fails to provide sufficient "Spatial Intelligence" at lower zoom levels.
- **Recommendation:** Increase `weight` to `3.5` and `opacity` to `0.4` for active lines.

### 2. Background Contrast & Environmental Shading
- **Status:** **POOR**
- **Finding:** The global CSS filters on `.retro-map .leaflet-container` (brightness 0.85, contrast 1.2, saturate 0.8) are washing out the subtle green of parks and blue of the Thames. 
- **Evidence:** `frontend/src/index.css`
```css
.retro-map .leaflet-container {
  background: #0b0b0d !important;
  filter: brightness(0.85) contrast(1.2) saturate(0.8);
}
```
- **Recommendation:** Remove or significantly tune the CSS filters. Rely on the "Dark Matter" tile set's native colors. Increase contrast of the GeoJSON layers instead of the whole container.

---

## Final Instructions for Frontend Engineer
1. **Fix Scroll Lock:** Add `document.body.style.overflow = 'auto';` to the `PreviewDrawer` cleanup function.
2. **Fix Sidebar Flicker:** Add GPU acceleration styles to the sidebar `aside` element.
3. **Refine Map:** Increase GeoJSON line weight/opacity and remove aggressive global CSS filters from the map container.
