# UI/UX & Data Fidelity Audit: 2026-03-08

## Audit Objective
Rigorous verification of the immoSearch dashboard against `REQUIREMENTS.md` and the "Linear meets Bloomberg" aesthetic mandate.

## 1. Requirement Compliance Matrix
| Requirement | Status | Observations |
| --- | --- | --- |
| **1. Data Integrity & Images** | ❌ FAILED | Frontend components (`Dashboard.tsx`, `PreviewDrawer.tsx`) are using hardcoded Unsplash URLs instead of the `image_url` field from `master.json`. This is a violation of the "Placeholder Prohibition" (Req 1.3). |
| **1. Running Costs** | ❌ FAILED | `service_charge`, `ground_rent`, and `lease_years_remaining` are missing from the `Property` type definition and are not rendered in the Detail/Preview views. |
| **2. Sorting Engine** | ⚠️ PARTIAL | Sorting exists but doesn't yet support multi-factor sorts like "Value Gap" or "Commute Utility" as specified in Req 2.3. |
| **5. Map Metro Overlay** | ❌ FAILED | London Metro GeoJSON overlay is not implemented (Req 5.1). |
| **8. Institutional Tooltips** | ⚠️ PARTIAL | Tooltips are brief and lack the methodology/strategic context required by Req 8.1. |
| **9. Market Pulse** | ✅ PASSED | Macro trends correctly visualized in `MarketPulse.tsx` and `MarketSituationRoom.tsx`. |

## 2. Aesthetic Audit: "Linear meets Bloomberg"
- **Linear alignment:** Precision typography and borders are excellent. Sidebar and Layout follow the `linear.app` pattern perfectly.
- **Bloomberg alignment:** Data density is high. Map markers and the "Pulse" header give a pro-tool feel.
- **Issue:** The "Protocol Overview" button on the Landing Page is non-functional.
- **Issue:** Hardcoded placeholder images break the "Institutional" immersion.

## 3. Data Fidelity Audit (master.json)
- **Link Integrity:** 30% of properties (e.g., Peters Court, Cavendish Mansions) are missing direct agent links. They rely exclusively on portals, which violates Req 1.11.
- **Schema Sync:** The `links` array is present in all entries, but the frontend is inconsistent in its usage.
